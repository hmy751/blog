# Source Policy

이 문서는 블로그 글쓰기 작업에서 원천 자료를 읽고 공개 가능한 글로 바꿀 때의 경계다.

블로그 글은 여러 source에서 출발할 수 있다. 프로젝트 코드, 작업 로그, 기존 글, 공식 문서, 회사 작업 회고, 개인 프로젝트 메모, PI Lab/dev-hub 자료 모두 source가 될 수 있다. 단, 공개 글에는 source의 문장이 아니라 공개 가능한 해석과 근거만 남긴다.

## 기본 원칙

- 원천 자료는 evidence다. 공개 블로그 repo는 writing/publishing surface다.
- 원천 자료의 문장을 가져오는 것이 아니라, 원천 자료를 근거로 공개 가능한 문장을 새로 쓴다.
- 내부 경로, 파일명, 라인 번호, 비공개 코드 원문, 면접 피드백 원문, 개인 메모 문장은 공개 글에 드러내지 않는다.
- 사실을 줄이거나 바꾸지 않는다. 공개할 수 없는 부분은 일반화하되, 일반화 때문에 의미가 바뀌면 쓰지 않는다.
- 코드 노출을 피하려다가 기술 주장이 검증 불가능해지면 멈추고 주장 범위를 줄인다.

## Source 종류

| Source | 사용 방식 | 주의 |
| --- | --- | --- |
| 기존 발행 글 | 흐름, 톤, 과거 주장 확인 | 과거 글의 표현을 무비판적으로 반복하지 않는다. |
| 프로젝트 코드/작업 로그 | 구현 사실, 기술 선택, 문제 상황 확인 | 내부 경로, 파일명, 라인 번호, 코드 원문 노출 금지. |
| 회사 작업 회고 | 문제, 제약, 개선 방향, 협업 맥락 확인 | 회사 내부 정보와 개인 평가가 드러나지 않게 일반화. |
| 개인 프로젝트 메모 | 제품 문제, 사용자 흐름, 설계 결정 확인 | 개인 메모 문장을 그대로 옮기지 않는다. |
| 공식 문서/외부 레퍼런스 | 기술 용어와 API 동작 확인 | 최신성 확인이 필요한 주장은 공식 문서를 우선한다. |
| PI Lab/dev-hub 자료 | 학습/실험 글의 수치, 사건, 판단 확인 | sprint 정리 문장과 미션 코드 세부를 직접 노출하지 않는다. |

## 공개 글에 직접 쓰지 않는 것

- 로컬 절대 경로와 개인 작업 디렉토리
- 내부 미션 코드 경로
- 비공개 코드 원문, 함수 본문, 파일별 구현 세부
- 면접 피드백 원문이나 개인 평가 문장
- 개인 메모의 날것 문장
- AI가 정리한 자료의 문장 그대로
- 회사 내부 사정, 담당자 평가, 비공개 지표

## 안전한 변환 방식

| 원천 | 공개 글 변환 |
| --- | --- |
| 내부 파일 경로 | "평가 시스템 코드", "실험 로그", "복습 노트", "작업 기록" |
| 코드 원문 | 짧은 일반화 예시 또는 개념 설명 |
| 회사 내부 맥락 | 공개 가능한 문제/제약/결정 기준 |
| 면접 피드백 | 글의 배경 판단 기준으로만 반영 |
| 개인 메모 | 사실 확인 후 블로그 문체로 재작성 |
| sprint 정리 문장 | 사실 확인 후 블로그 문체로 재작성 |
| 수치/결과 | 원천과 같은 값으로만 사용 |

## 글 유형별 경계

- 기술 구현/복기 글: 코드와 API 세부는 독자가 이해할 만큼만 일반화한다.
- 회사 프로젝트 글: 회사명이나 서비스명을 쓸 수 있어도 내부 프로세스, 담당자, 비공개 수치는 노출하지 않는다.
- 제품/아키텍처 글: 사용자 문제와 설계 결정은 공개하되 개인 메모나 내부 TODO를 그대로 옮기지 않는다.
- 개인 회고 글: 개인 감정과 고민은 쓸 수 있지만 타인의 평가나 비공개 피드백을 끌어오지 않는다.
- 학습/실험 글: 실험 조건과 수치는 정확히 쓰되 내부 미션 코드 세부와 source path는 숨긴다.

## 멈춤 신호

- 내부 자료 문장을 붙여넣고 있다.
- 공개할 수 없는 세부를 "조금만" 바꿔 쓰고 있다.
- 글의 재미를 위해 실제 배운 범위가 좁아졌다.
- 코드 노출을 피하려다 기술 주장이 검증 불가능해졌다.
- 회사/동료/평가 맥락이 글의 근거가 아니라 노출 위험으로 보인다.
- source보다 글이 더 구체적인 수치나 시점을 말하고 있다.

이 신호가 보이면 초안 작성을 멈추고 원천 카드, 공개 가능한 주장, 삭제할 세부를 다시 분리한다.
