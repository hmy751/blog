# Source Policy

이 문서는 블로그 글쓰기 작업이 `dev-hub`와 PI Lab 자료를 읽을 때의 경계다.

## 기본 원칙

- `dev-hub`는 source/evidence다. 공개 블로그 repo는 writing/publishing surface다.
- 원천 자료의 문장을 가져오는 것이 아니라, 원천 자료를 근거로 공개 가능한 문장을 새로 쓴다.
- 공개 글에서는 내부 경로, 파일명, 라인 번호, 미션 코드 원문, 면접 피드백 원문을 드러내지 않는다.
- 사실을 줄이거나 바꾸지 않는다. 공개할 수 없는 부분은 일반화하되, 일반화 때문에 의미가 바뀌면 쓰지 않는다.

## 읽을 수 있는 자료

- `~/Desktop/10_work/dev-hub/track/pilab/`의 sprint 정리, 복습노트, 회고 메모
- `~/Desktop/10_work/dev-hub/track/pilab/blog/메모/`의 원천 메모
- 기존 공개 블로그 글과 발행 후보 초안
- 사용자가 명시한 추가 자료

## 공개 글에 직접 쓰지 않는 것

- `/Users/...`, `~/Desktop/...` 같은 로컬 절대 경로
- `pilab/missions/...` 같은 내부 미션 코드 경로
- 실제 코드 원문, 함수 본문, 파일별 구현 세부
- 면접 피드백 원문이나 개인 평가 문장
- AI가 정리한 sprint 자료의 문장 그대로

## 안전한 변환 방식

| 원천 | 공개 글 변환 |
| --- | --- |
| 내부 파일 경로 | "평가 시스템 코드", "실험 로그", "복습 노트" |
| 코드 원문 | 짧은 일반화 예시 또는 개념 설명 |
| 면접 피드백 | 글의 배경 판단 기준으로만 반영 |
| sprint 정리 문장 | 사실 확인 후 블로그 문체로 재작성 |
| 수치/결과 | 원천과 같은 값으로만 사용 |

## 멈춤 신호

- 내부 자료 문장을 붙여넣고 있다.
- 공개할 수 없는 세부를 "조금만" 바꿔 쓰고 있다.
- 글의 재미를 위해 실제 배운 범위가 좁아졌다.
- 코드 노출을 피하려다 기술 주장이 검증 불가능해졌다.

이 신호가 보이면 초안 작성을 멈추고 원천 카드, 공개 가능한 주장, 삭제할 세부를 다시 분리한다.
