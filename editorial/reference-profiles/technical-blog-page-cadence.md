# Technical Blog Page Cadence

이 문서는 특정 글의 디자인을 복제하기 위한 기준이 아니다. 기술 글이 산문만으로 밀리지 않고, 독자가 스크롤하면서 질문·근거·판단을 복구하게 만드는 페이지 리듬 패턴이다.

## 핵심 패턴

- 초반 3~5문단 안에 글의 질문, 범위, 읽는 관점을 잡는다.
- 큰 파트 제목은 글의 단계나 질문 전환을 보여주고, 작은 판단 소제목은 그 구간에서 얻은 판단을 보여준다.
- 긴 산문은 `claim -> artifact -> recovery` 흐름으로 끊는다.
- 화면으로 판단해야 하는 재료는 표나 텍스트 도식으로만 축소하지 않는다.
- 자료 앞에는 무엇을 봐야 하는지 열고, 자료 뒤에는 다음 판단 한 문장만 남긴다.
- before/after가 핵심이면 이전 상태와 바뀐 상태를 독자가 나란히 복구할 수 있게 한다.
- 세 화면 이상 같은 밀도의 산문이 이어지면 소제목, 짧은 리스트, 표, 코드/의사코드, 도식, 캡션, 콜아웃 중 하나로 복귀 지점을 만든다.
- 소제목이 달라져도 본문 단락이 같은 전개로 닫히면, 섹션별 단락 역할을 다시 배치한다.
- 자료 장치는 글감을 대체하지 않고, 이미 살아 있는 판단을 독자가 놓치지 않게 돕는다.

## Page Cadence Map

레퍼런스나 페이지 리듬 요청이 강한 편집에서는 먼저 아래 map을 만든다.

| 구간 | 독자가 잡아야 할 판단 | 현재 원고의 증거 | 필요한 장치 | 실제 move |
| --- | --- | --- | --- | --- |
| opening | 글의 질문과 읽는 관점 | 제목/도입 문단 | 대표 장면, 짧은 약속 | 앞으로 당길 문단 또는 새 lead |
| first body | 문제의 구체 모습 | 실패, 출력, trace, 수치 | before/after, screenshot, 표 | 삽입/슬롯/대체 도식 |
| middle | 판단 변화 | 비교, 실험, 구현 결정 | 리스트, 표, 코드, 회수 문장 | 산문 분리 또는 자료화 |
| section close | 다음 절로 가져갈 기준 | 섹션 끝 문단 | 짧은 회수 | 1문장 축소 |
| ending | 글의 발견 | 마지막 문단 | 기준 질문 | 발견 회수 |

map은 진단표가 아니라 편집판이다. edit mode라면 이 map에서 실제 원고 move나 후보 슬롯으로 이어지는 항목을 고른다.

## Body Cadence Map

본문 리듬 자체가 쟁점이면 page cadence map 다음에 섹션별 단락 역할을 본다. 목적은 모든 섹션을 다르게 꾸미는 것이 아니라, 같은 산문 엔진이 반복되어 글의 발견이 평평하게 들리는 구간을 찾는 것이다.

| 섹션 | 현재 여는 방식 | 현재 닫는 방식 | 실제 단락 역할 | 바꿀 move |
| --- | --- | --- | --- | --- |
| opening | 배경 설명 / 질문 / 장면 | 글의 약속 | question / scene | 대표 질문을 앞으로 당기거나 범위 약속을 줄인다 |
| first body | 설명 / 실패 장면 | 원인 후보 | scene / evidence | 실패 샘플, trace, 숫자, before/after를 먼저 보인다 |
| middle | 기술 설명 | 판단 회수 | mechanism / decision | 산문 일부를 표, 코드, 리스트, 캡션으로 나눈다 |
| later body | 또 다른 설명 | 같은 결론 반복 | consequence / carry-forward | 다음 섹션으로 가져갈 질문만 남긴다 |
| ending | 요약 | 발견 회수 | discovery | 반복된 결론을 줄이고 마지막 기준 질문을 남긴다 |

진단할 반복 신호:

- `처음에는`, `하지만`, `그래서`, `중요한 것은`, `결국`이 여러 섹션에서 같은 위치에 반복된다.
- 모든 섹션이 `오해 -> 설명 -> 재해석`으로 닫힌다.
- 표나 코드 뒤에도 같은 결론을 산문으로 다시 길게 설명한다.
- 독자가 볼 수 있는 장면보다 글쓴이의 정리 문장이 먼저 나온다.

edit mode에서는 map을 만든 뒤 최소 2개 섹션의 역할을 바꾼다. 예: 첫 본문은 실패 장면으로 열고, 중간 섹션은 비교표가 판단을 먼저 보여주게 하고, 후반 섹션은 짧은 선택지나 rollback 질문으로 시작한다.

## 복제 금지

- 특정 레퍼런스의 디자인, 히어로 이미지, 소제목 개수, 표 개수, 문체를 따라 하지 않는다.
- 플랫폼 화면 폭이나 캡처 파일의 구성을 active guide로 만들지 않는다.
- 페이지 리듬을 이유로 모든 기술 글을 튜토리얼이나 문서형 글로 평평하게 만들지 않는다.
- 자료 장치를 넣었다는 이유로 중심 질문, 발견, texture 점검을 생략하지 않는다.
