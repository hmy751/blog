# 편집 패턴 메모

> 특정 글의 1회성 교정이 아니라 **반복된 패턴**만 모은다. 사례가 3회 이상 쌓인 것만 등재.
> 이 문서는 완결된 법전이 아니라 작업 중 누적되는 관찰 로그.
> 글 유형별로 적용 강도가 다르다. 특히 회고 글에는 자기 관찰과 했다체가 자연스러운 경우가 있으므로, 기술 글의 금기를 그대로 적용하지 않는다.

---

## 1. 담백함의 실제 기준

담백함 = **호흡이 있는 솔직함 + 구체 장면**. 특히 기술 글과 학습/실험 글에서 다음 표현은 담백이 아님.

| 거부된 표현 | 문제 | 바뀐 방향 |
|---|---|---|
| "그 관심은 곧 질문으로 바뀌었습니다" | 시적 은유 | 실제 장면 서술 |
| "곧 구체적인 질문이 생겼습니다" | 과잉 압축, 장면 없음 | 호흡 복원 (`직접 실험을 해보니, 그제야 ~`) |
| "이 질문들이 먼저였습니다" (단독 close) | 무미건조, 연결 끊김 | 자연스러운 다음 문단 연결 |
| "감각이었습니다" | 감상·자기성찰 과잉 | 사실 서술 |
| "분야로 보기 시작했습니다" | memoir-style 자기 관찰 | 상태 변화 서술 (`AI가 ~로 옮겨갔습니다`) |
| "다음 단계의 자연스러운 질문입니다" | formulaic 관용구 | 본인 의지 표현 (`확인해보고 싶습니다`) |

**원칙**: 압축으로 담백을 만들지 말 것, 감상으로 깊이를 만들지 말 것, 관용구로 마무리하지 말 것. 단, 회고 글에서는 자기 관찰 자체가 문제가 아니라 과장된 전환이나 근거 없는 감상이 문제다.

## 2. 팩트·전제 축소 금지

대비 프레임을 쓸 때 실제 범위·배경을 한 층으로 축소하는 표현 경계.

| 거부된 표현 | 문제 | 바뀐 방향 |
|---|---|---|
| "개발할 때와는 다른 질문" | FE 배경 무효화 (읽는 사람이 "이 사람 FE할 때 성능 관심 없었나"로 오해) | "같은 기준이 실험의 언어로 다시" (연속성) |
| "트랜스포머 구조 자체보다도" | 실제 배운 범위(데이터 전처리 → 선형 모델 → 손실 함수 → 트랜스포머 → 전이학습)를 한 층으로 축소 | 범위 열거 후 "그 안에서 ~" |
| "페어 두 분(저자 본인·파트너)" | 저자 본인을 제3자처럼 혼입 | "페어로" / "페어와 함께" |

**원칙**: 대비나 드라마를 위해 사실을 줄이지 않기.

## 3. 자기 드러내기 최소화

기술 글에서 선별·평가·관찰 주체로서 "저/제가"가 등장하면 점검. 단, 내용의 핵심이 본인 경험일 때는 유지. 회고 글에서는 1인칭이 자연스러울 수 있으므로, 1인칭 자체보다 자기 포장이나 반복을 본다.

| 경계 대상 | 유지 |
|---|---|
| "제가 특히 크게 배운 부분" (선별 주체) | "제가 틀렸다는 사실이 더 또렷해졌습니다" (경험 핵심) |
| "저는 이때부터 ~를 ~로 보기 시작했습니다" (관찰 주체) | "저는 이 문장을 처음 제대로 체감했습니다" (경험 핵심) |

**판정 기준**: **"내가 어떻게 느꼈다/봤다"는 경계 대상, "내가 무엇을 했다/틀렸다"는 유지**.

## 5. 구조 수준 공통 함정

- **하이라이트(bold/결론)가 본문에서 근거 없이 등장** — bold로 얹기 전에, 본문 어딘가에 구체 사례·에피소드가 있는지 확인
- **도입부 문단 순서가 실제 시간/논리 역순** — 결과를 먼저 선언한 뒤 과정으로 되돌아가면 흐름 끊김
- **같은 메타 반성이 두 문단 연속 등장** — 한 호흡으로 통합

## 6. sprint 자료 활용의 균형

이 섹션은 PI Lab 또는 학습/실험 글에 우선 적용한다.

- generic 기술 서술(교과서 문체)로 기울면 본인 경험의 질감이 죽는다
- 반대로 sprint 자료 원문을 그대로 전재하면 블로그 장르와 안 맞을 수 있다 (자료도 AI 정리가 섞여 있음)
- **재료로 참고**하되 **블로그 결에 맞게 다시 판단** — 중간 지점이 기본값
- 본인이 실제 쓴 비유(`16명 평균`, `오탈자까지 외우기`, `모델은 뭘 보고 예측하는가`)가 있으면 우선 활용
