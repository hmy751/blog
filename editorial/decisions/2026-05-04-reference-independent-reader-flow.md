# Reference-independent reader flow moves

작성일: 2026-05-04
상태: accepted

## 배경

`artifact gap` 가드를 추가한 뒤 초안은 산문만 이어지던 문제에서 어느 정도 벗어났다. LangSmith 실행 단위 글에도 책임 표, trace tree, 실행 경로 표, snapshot 표가 들어가며 독자가 볼 수 있는 근거가 늘었다.

하지만 하네스는 여전히 안전한 Markdown 표와 텍스트 도식으로 먼저 수렴하는 경향이 있었다. 사용자가 기대한 것은 실제 화면, 스크린샷, 이미지화된 도식, 캡션, 대표 장면, 전후 대비, 섹션 끝 회수, 스크롤 손잡이가 함께 작동하는 글이었다.

또 하나의 위험은 특정 참고 글이나 루트 캡처를 active guide 안에서 직접 전제하는 것이다. 참고 자료는 나중에 삭제되거나 이동할 수 있으므로, 반복 실행되는 하네스는 그 파일을 다시 볼 수 있어야만 작동해서는 안 된다.

## 기존 하네스가 놓친 문제

- `artifact gap`은 보이는 근거 부족을 잡았지만, 실제 스크린샷이나 이미지 후보를 우선 검토해야 하는 경우는 약했다.
- Reader Flow는 오프닝, 소제목, 문단 호흡을 보지만, 대표 장면, before/after 대비, 섹션 끝 회수, 스크롤 리듬을 독립 edit move로 충분히 부르지 않았다.
- Supporting Materials는 표, 도식, 스크린샷을 나란히 다뤘지만, trace UI나 제품 화면처럼 원래 화면으로 보는 재료도 표나 텍스트 도식으로 대체될 수 있었다.
- 참고 글의 효과를 일반화하지 않고 "레퍼런스처럼"이라고만 남기면, 해당 참고 자료가 사라졌을 때 기준을 재현하기 어렵다.

## 결정

새 단계는 만들지 않는다. 기존 Reader Flow와 artifact gap 처리 안에 아래 edit move를 추가한다.

- **대표 장면**: 초반이나 첫 본론에 질문, 출력, 실패, 로그, trace, 화면 상태처럼 글의 문제를 구체화하는 장면을 둔다.
- **before/after 대비**: 판단이 바뀐 글에서는 이전 모양과 바뀐 모양을 독자가 나란히 복구할 수 있게 한다.
- **섹션 끝 회수**: 각 절이 독립 설명으로 닫히지 않고, 다음 절로 들고 갈 판단을 한두 문장으로 남긴다.
- **스크롤 손잡이**: 긴 산문이 이어질 때 소제목, 짧은 리스트, 표, 코드/의사코드, 도식, 캡션, 콜아웃 같은 작은 복귀 지점을 검토한다.
- **visual candidate**: trace UI, 제품 화면, 모델 답변, 그래프처럼 화면성이 강한 재료는 표나 텍스트 도식으로만 끝내지 않고 스크린샷/이미지 후보를 먼저 검토한다.
- **후보 슬롯**: 스크린샷/이미지를 추천했지만 파일, 캡처 권한, 마스킹 범위가 없어 바로 삽입할 수 없으면 최종 답변에서만 언급하지 않고 원고의 해당 위치에 `supporting-material candidate` 주석을 남긴다.

참고 글이나 캡처가 있을 때는 파일 자체를 active dependency로 남기지 않는다. 관찰한 효과만 `대표 장면`, `before/after`, `섹션 회수`, `스크롤 손잡이`, `visual candidate` 같은 현재 글의 edit move로 번역한다.

## 적용 범위

- PI Lab 기술/실험 글
- LangSmith/RAG/evaluation/config/UI/trace/그래프처럼 화면성이 있는 artifact가 핵심인 글
- 사용자가 "가독성", "읽기 좋게", "좋은 글처럼", "레퍼런스처럼", "이미지", "스크린샷", "도표", "호흡"을 언급한 편집 요청

## 비목표

- 모든 글에 히어로 이미지나 스크린샷을 강제하지 않는다.
- 분위기용 이미지 생성을 기본값으로 만들지 않는다.
- 공개 경계가 불확실한 화면을 그대로 노출하지 않는다.
- 후보 슬롯을 발행 원고에 남겨두지 않는다. 발행 전에는 실제 자료로 바꾸거나 제거한다.
- 참고 글의 레이아웃이나 디자인을 복제하지 않는다.
- 참고 파일이 없어지면 작동하지 않는 규칙을 active guide에 두지 않는다.
- 표나 텍스트 도식이 더 나은 경우까지 시각 자료로 대체하지 않는다.

## 근거

LangSmith 실행 단위 초안은 artifact gap 보강 뒤에도 실제 trace 화면 후보가 최종 답변에서만 언급되고 원고에는 남지 않는 문제가 있었다. 텍스트 trace tree는 실행 구조를 설명하지만, LangSmith UI에서 질문 하나가 root run으로 묶이는 장면을 독자가 직접 확인하게 하지는 못했다.

사용자 피드백도 같은 방향이었다. 좋은 참고 글은 실제 스크린샷, 도표, 캡션, 문단 호흡, 중간 손잡이를 함께 쓰는데, 하네스는 "자료 장치는 기본값이 아니다"라는 가드 때문에 안전한 표와 텍스트 도식까지만 만들고 멈추는 경향이 있었다.

다만 참고 글이나 루트 캡처 파일을 active guide에 직접 의존시키면 해당 파일이 삭제되거나 이동했을 때 기준을 재현할 수 없다. 따라서 관찰한 효과를 현재 글에 적용 가능한 edit move와 후보 슬롯 규칙으로 번역하는 쪽이 더 안정적이다.

## 후속 점검

- Shaping Editor의 `Reader / Artifact gap`에 화면성이 강한 재료의 visual candidate가 실제로 남는지 확인한다.
- Reader Flow 적용 결과가 대표 장면, before/after, 섹션 회수, 스크롤 손잡이 같은 edit move로 이어지는지 확인한다.
- Texture Keeper가 visual candidate나 리듬 장치가 글감을 대체하지 않는지, 반대로 안전한 표/텍스트 도식으로만 축소해 글이 빈약해지지 않는지 확인한다.
- Supporting Materials가 visual-first를 이유로 장식 이미지 체크리스트로 비대해지지 않는지 확인한다.
