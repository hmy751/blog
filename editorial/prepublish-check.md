# Prepublish Check

발행 전 마지막 확인 목록이다. `content/posts/`로 승격하기 전에 적용한다.

## Frontmatter

- `title`이 독자가 보는 제목과 일치한다.
- `date`가 실제 발행일이며 `TBD`가 아니다.
- `description`이 있다.
- 임시 상태값, 작업 메모, 내부 태그가 남아 있지 않다.

## 공개 경계

- 로컬 절대 경로가 없다.
- 내부 미션 코드 경로가 없다.
- PI Lab 코드 원문이나 내부 파일명이 노출되지 않는다.
- 면접 피드백, 개인 평가, private memo 문장을 그대로 쓰지 않았다.
- Claude/Codex 생성 흔적이 없다.

## 글 품질

- bold나 결론 문장이 본문에서 구체 근거를 받는다.
- 도입부가 실제 시간/논리 순서와 어긋나지 않는다.
- 같은 메타 반성이 두 문단 이상 반복되지 않는다.
- 담백함을 만들려고 장면을 과하게 압축하지 않았다.
- 수치, 용어, 시점, 사람 이름이 원천 자료와 맞는다.

## 실행

```sh
node scripts/blog-prepublish-check.mjs
```

이 스크립트는 deterministic guard다. 통과해도 글이 좋은 것은 아니며, 실패하면 공개 경계나 발행 메타데이터를 먼저 고친다.
